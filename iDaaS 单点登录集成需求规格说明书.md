# iDaaS单点登录集成需求规格说明书
## 文档版本：V1.0
## 适用范围：公司内部Java Web应用系统IDaaS单点登录集成

# 1 文档概述
## 1.1 文档目的
本需求规格用于规范公司内部Java Web应用系统与**IDaaS统一单点登录认证平台**的集成标准，明确集成的基础约束、技术要求、实施步骤，确保应用系统预留标准化集成接口，在公司内网部署后可快速、合规完成IDaaS单点登录对接，实现统一身份认证管控。

## 1.2 核心集成目标
Web应用系统登录时调用IDaaS统一登录账号与登录页面，通过IDaaS SDK完成单点登录认证，替代应用本地独立登录逻辑。

# 2 基础集成要求
## 2.1 集成核心要求
IDaaS为公司内部统一单点登录认证平台，本次开发的Web应用系统**必须预留标准化IDaaS集成接口**，无需提前完成全量集成开发，仅需按规范完成代码、配置、接口预留，确保后续内网部署后可快速对接上线。

## 2.2 技术栈约束
仅支持**Java Web应用**集成；IDaaS SDK基于`javax.servlet.Filter`实现请求过滤，**集成应用必须支持Servlet API**。

## 2.3 域名约束
Web应用必须使用**引望官方域名**或IDaaS支持的域名（`.yinwang.com`）与IDaaS平台进行交互，禁止使用自定义私有域名对接。

## 2.4 运行环境约束
应用运行环境**仅支持JDK1.8及以上版本**，低版本JRE无法兼容IDaaS SDK。

## 2.5 网络环境约束
Web应用服务器**必须部署在引望内网环境**，外网环境不支持IDaaS单点登录集成。

## 2.6 认证凭证管理
1. 各应用需**自行申请独立的AKSK和APPID**，严禁盗用、共用其他应用的APPID及对应AKSK；
2. 申请后的AKSK、APPID等敏感信息需妥善保管，禁止明文泄露、代码硬编码等违规操作。

# 3 Web项目SDK集成实施要求
本章节为应用预留集成接口的标准化实施要求，所有Java Web项目需按以下规范完成SDK集成代码、配置预留。

## 3.1 项目基础环境准备
1. 基于标准Java Web项目开发，推荐SpringBoot架构；
2. 参考Demo标准运行环境：
   - JDK版本：`1.8.0_332`
   - SpringBoot父依赖版本：`spring-boot-start-parent 2.7.12`
3. 项目需完整支持Servlet API，满足Filter过滤器运行条件。

## 3.2 Maven依赖引入
项目需在`pom.xml`中预留IDaaS SDK Maven依赖配置节点，确保依赖可正常引入：
```xml
<!-- IDaaS单点登录SDK依赖（预留配置） -->
<dependency>
    <groupId>com.yinwang</groupId>
    <artifactId>idaas-sso-sdk</artifactId>
    <version>最新稳定版</version>
</dependency>
```

## 3.3 SSO过滤器配置
通过**Spring配置类**完成IDaaS官方`SsoFilter`的注册与过滤规则配置，作为核心SSO认证过滤器预留：
1. 创建过滤器配置类，注册IDaaS SDK自带的`SsoFilter`；
2. 配置过滤器拦截规则，拦截应用所有登录相关请求，转发至IDaaS认证；
3. 配置过滤器执行顺序，保证`SsoFilter`优先于自定义过滤器执行。

## 3.4 自定义登录登出逻辑开发
应用需按规范自定义登录、登出业务逻辑，完成过滤器与接口预留：
1. **自定义登录过滤器**：创建`LoginFilter`登录控制过滤器，实现IDaaS认证通过后的本地会话创建、用户信息同步等逻辑；
2. **过滤器注册**：将自定义`LoginFilter`注册到项目中，配置合理的拦截顺序；
3. **退出逻辑实现**：开发单点登出接口，对接IDaaS登出能力，实现应用本地会话销毁+IDaaS全局会话注销的统一登出逻辑。

# 4 集成约束与规范
1. 所有集成代码需遵循公司Java开发规范，禁止修改IDaaS SDK底层源码；
2. 集成预留代码需添加清晰注释，便于后续对接调试；
3. 敏感配置（AKSK、APPID、IDaaS平台地址）需采用配置文件注入，禁止硬编码。

# 5 验收标准
1. 项目完成所有基础约束适配（JDK、Servlet、域名、内网环境）；
2. 已预留IDaaS SDK Maven依赖、SSO过滤器、自定义登录登出过滤器；
3. 集成代码符合规范，无敏感信息硬编码，可直接对接IDaaS平台完成上线。

---

### 总结
1. 核心约束：仅支持**Java Web+JDK1.8+Servlet API+引望内网+yinwang域名**的集成环境；
2. 实施要求：预留SDK依赖、官方SSO过滤器、自定义登录登出逻辑；
3. 安全要求：独立申请AKSK/APPID，严禁盗用与明文泄露。
